# 详细设计-05-导师工作台

## 1. 模块目标

导师工作台用于导师查看学生档案、阅读资料摘要、确认 SOP、维护任务、处理目标修订和记录沟通情况。

导师是 AI 建议和学生行动之间的人工校准者。系统不能让 AI 直接替代导师确认。

## 2. 用户范围

导师端主要面向：

- 保研导师。
- 考研导师。
- 就业导师。
- 竞赛导师。
- 科研导师。
- 出国和考公方向导师。

MVP 阶段导师端可先适配桌面和平板，不要求深度移动端体验。

## 3. 核心页面

### 3.1 待处理学生列表

字段：

- 学生姓名。
- 学校。
- 专业。
- 目标标签。
- 当前状态。
- 资料解析状态。
- SOP 状态。
- 最近更新时间。

筛选：

- 待确认 SOP。
- 待首次沟通。
- 目标修订待处理。
- 高风险学生。

### 3.2 学生档案详情

展示：

- 基础信息。
- 目标标签。
- 上传资料列表。
- Dify 文件摘要。
- AI 识别的关键规则。
- 风险项。
- 当前 SOP。
- 沟通记录。

### 3.3 SOP 审核与确认

导师操作：

- 查看 AI V0.1 草案。
- 修改四年路线图。
- 调整任务。
- 增加红线预警。
- 添加导师备注。
- 确认发布 V1.0。

### 3.4 任务维护

导师操作：

- 新增任务。
- 修改任务。
- 调整优先级。
- 设置截止时间。
- 标记风险。

### 3.5 目标修订审核

导师操作：

- 查看学生修订原因。
- 查看 Dify 影响分析。
- 沟通后确认或驳回。
- 确认后生成新 SOP 版本。

## 4. 业务流程

### 4.1 首次确认 SOP

```text
导师打开待处理学生
  ↓
查看学生资料和 AI 摘要
  ↓
查看 SOP V0.1 草案
  ↓
调整路线图和任务
  ↓
确认 SOP V1.0
  ↓
系统通知学生再次碰卡查看
```

### 4.2 目标修订处理

```text
学生提交目标修订
  ↓
Dify 生成影响分析
  ↓
导师查看修订申请
  ↓
导师沟通确认
  ↓
生成新 SOP 版本
  ↓
旧 SOP 归档
```

## 5. 后端接口设计

### 5.1 获取导师学生列表

```text
GET /api/mentor/students
```

查询参数：

```text
status=
keyword=
goal=
page=
page_size=
```

### 5.2 获取学生详情

```text
GET /api/mentor/students/{student_id}
```

### 5.3 获取待确认 SOP

```text
GET /api/mentor/students/{student_id}/sop-draft
```

### 5.4 编辑 SOP

```text
PATCH /api/mentor/sops/{sop_id}
```

### 5.5 确认 SOP

```text
POST /api/mentor/sops/{sop_id}/confirm
```

### 5.6 新增沟通记录

```text
POST /api/mentor/students/{student_id}/communication-logs
```

### 5.7 处理目标修订

```text
POST /api/mentor/goal-revisions/{revision_id}/review
```

## 7. 权限规则

- 导师只能查看分配给自己的学生。
- 导师不能查看其他导师学生的私有资料。
- 导师可以编辑自己学生的 SOP 草案。
- 导师确认 SOP 后，需要记录操作日志。
- 导师不能删除学生历史 SOP。

## 9. 异常处理

| 场景 | 处理 |
| --- | --- |
| 导师访问未分配学生 | 返回 403 |
| SOP 已确认后重复确认 | 返回业务错误 |
| Dify 摘要缺失 | 展示暂无摘要，允许导师人工确认 |
| 目标修订资料不足 | 允许导师驳回并要求补充 |
| 通知发送失败 | SOP 照常确认，记录通知失败并重试 |

## 10. 验收标准

- 导师能看到自己分配的学生。
- 导师不能访问未分配学生。
- 导师能查看学生上传资料摘要。
- 导师能编辑并确认 SOP。
- 确认 SOP 后学生再次碰卡进入仪表盘。
- 目标修订能由导师审核并生成新 SOP。
- 沟通记录能保存并在学生详情中查看。

## 11. V2.4 导师闭环修订

导师工作台首页待办按时效排序：

1. 已逾期且未提交的高优先级任务。
2. 待验收成果。
3. 被退回后长期未补交的任务。
4. 待反馈复盘。
5. 待确认 SOP 和目标修订。
6. 待首次沟通和普通消息。

新增页面：

- 成果验收队列：预览证据、通过、退回、查看历史版本。
- 风险学生列表：按红线、连续延期、长期无行动、情绪自评等来源分组。
- 周期复盘页：查看系统汇总，填写导师反馈，发布下一周期重点。
- 成长档案：查看已验收成果、荣誉、奖金状态和目标变化时间线。

新增接口：

```text
GET  /api/mentor/task-submissions
GET  /api/mentor/task-submissions/{id}
POST /api/mentor/task-submissions/{id}/accept
POST /api/mentor/task-submissions/{id}/request-changes
GET  /api/mentor/reviews?status=submitted
POST /api/mentor/reviews/{id}/feedback
POST /api/mentor/students/{id}/risk-interventions
```

导师退回成果必须填写原因；通过涉及荣誉或奖金的成果时，必须选择“仅任务验收”或“同时提交荣誉/奖金审核”，防止任务验收直接产生未经审核的财务结果。
