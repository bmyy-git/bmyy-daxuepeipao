# 详细设计-07-学生全周期闭环

## 1. 模块目标

本模块把学生从 NFC 激活到毕业成长档案串成一个可执行、可验收、可复盘、可修订的长期闭环。学生端不是静态看板，核心是持续回答四个问题：

1. 我现在最该做什么？
2. 我需要提交什么才算完成？
3. 导师如何反馈，我下一步如何改进？
4. 我的行动如何沉淀为规划版本、成长记录、荣誉和奖金状态？

## 2. 总体流程

```text
激活建档
  → 资料解析
  → SOP V0.1
  → 导师匹配与首次沟通
  → 合作模式确认
  → SOP V1.0 生效
  → 周任务发布
  → 学生执行与成果提交
  → 导师验收或退回
  → 成长事件、风险与通知更新
  → 周/月/学期复盘
  → 下一周期任务
  → 目标修订与新 SOP
```

## 3. 学生行动中心

`GET /api/students/me/action-center` 返回统一待办，不由前端分别拼接：

```json
{
  "stage": "active",
  "sop_version": "V1.0",
  "urgent_actions": [],
  "today_tasks": [],
  "waiting_reviews": [],
  "mentor_feedback": [],
  "risk_alerts": [],
  "next_review_at": "2026-06-21T12:00:00+08:00"
}
```

排序规则：

```text
被退回待补交 > 已逾期 > 24 小时内截止 > 导师新反馈 >
今日任务 > 待复盘 > 普通任务
```

## 4. 任务详情与成果提交

任务必须包含：

- 任务目的和关联目标。
- 完成标准。
- 截止时间和优先级。
- 资料来源与导师备注。
- 允许的成果类型。
- 当前提交版本和验收状态。

成果类型：

```text
text / image / document / link / checklist / metric
```

提交规则：

- 学生可保存草稿。
- 正式提交后任务进入 `submitted`。
- 导师处理前学生可撤回，撤回保留记录。
- 退回后创建新版本，不覆盖旧证据。
- 文件使用私有存储和短期预览链接。

## 5. 导师验收

导师决策：

| 决策 | 结果 |
| --- | --- |
| 通过 | 任务进入 `accepted`，写成长事件 |
| 退回补充 | 任务进入 `changes_requested`，通知学生 |
| 风险升级 | 额外创建风险干预记录，可通知项目老师 |

验收通过可附带：

- 导师评语。
- 能力标签。
- 成长积分。
- 荣誉申报建议。
- 奖金审核建议。

## 6. 周期复盘

学生复盘字段：

- 本周期最重要的完成事项。
- 未完成原因。
- 最大困难。
- 状态评分和压力评分。
- 希望导师支持的事项。
- 下一周期目标建议。

系统自动汇总：

- 任务验收率和延期率。
- 被退回次数。
- 连续行动天数。
- 红线状态。
- 新增成果和荣誉。
- AI 问答高频主题。

导师反馈后关闭本期复盘，并确认下一周期任务。未关闭的复盘不影响日常任务执行，但必须进入双方待办。

## 7. 风险闭环

风险来源：

- 红线规则命中。
- 高优先级任务逾期。
- 连续多个周期低完成率。
- 学生主动报告阻塞或高压力。
- 导师人工标记。
- AI 仅可建议风险，不可自动作出纪律或财务结论。

风险状态：

```text
open → acknowledged → intervening → resolved
                         ↓
                      escalated
```

每个风险必须记录负责人、处理动作、下次跟进时间和关闭说明。

## 8. 目标修订

学生发起修订时，当前 SOP 继续有效。流程为：

```text
revision.draft
  → submitted
  → ai_analyzed
  → mentor_reviewing
  → approved / rejected / needs_discussion
  → new_sop_confirmed
```

新 SOP 确认时：

1. 旧 SOP 归档。
2. 未完成任务按导师选择迁移、取消或重建。
3. 新版本生效。
4. 写入成长事件和通知。

## 9. 成长、荣誉与奖金

成长档案只收录可追溯事件：

- SOP 生效与版本变化。
- 任务验收通过。
- 周期复盘关闭。
- 荣誉审核通过。
- 奖金状态变化。
- 导师重要评价。

荣誉和奖金必须有独立审核状态，不因前端打卡自动产生。奖金使用不可变流水，展示应计、审核、待支付、已结算和驳回原因。

## 10. 媒体与体验

- 学生首次进入使用 `/mp4/yaya-smart.mp4`，静音自动播放并允许跳过。
- 视频失败或减少动态效果时回退 `/img/yayasmart-logo-1-1152-928.png`；该文件由原始特殊字符文件名在构建时规范化生成。
- 仪表盘和成长页使用 `/img/yayasmart-logo-3A-1280-1080.png` 等横向图作低对比度场景视觉。
- 空状态使用 `/img/yayasmart-logo-2A-900-900.png` 或 `2B`，同时提供文字下一步。
- 所有媒体均懒加载，不得遮挡任务和提交按钮。

## 11. 通知矩阵

| 事件 | 学生 | 导师 | 家长 |
| --- | --- | --- | --- |
| 导师分配 | 短信/邮件/站内 | 站内 | 不通知 |
| 任务临期 | 站内，可选短信 | 汇总 | 不通知 |
| 成果提交 | 状态回执 | 站内待办 | 不通知 |
| 成果退回/通过 | 站内，重要任务可短信 | 状态回执 | 仅授权的成长摘要 |
| 复盘提交 | 状态回执 | 站内待办 | 不通知 |
| 荣誉通过 | 站内 | 站内 | 授权后可见 |
| 家长鼓励 | 站内 | 不通知 | 发送回执 |

## 12. 验收标准

- 学生不能通过勾选直接完成需导师验收的任务。
- 每次成果提交和导师反馈均有版本与时间戳。
- 被退回任务可补交，历史记录完整。
- 行动中心能聚合学生所有紧急待办。
- 周期复盘能生成导师反馈和下一周期任务。
- 风险从发现到关闭有负责人和处理记录。
- 目标修订不覆盖旧 SOP。
- 荣誉和奖金仅由已验收、可审核的数据产生。
- 家长只能看到学生授权的成长摘要。
- 页面实际引用根目录 `img` 和 `mp4` 对应的生产静态资源路径。
