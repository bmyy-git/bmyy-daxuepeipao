# 详细设计-06-权限与隐私

## 1. 模块目标

权限与隐私模块负责保护学生个人资料、上传文件、SOP、问答日志和导师工作内容，防止越权访问和敏感信息泄露。

本系统涉及手机号、邮箱、学校资料、学生目标、问答记录和可能的成绩相关信息，必须从 MVP 阶段就建立基本权限边界。

## 2. 用户角色

| 角色 | 说明 |
| --- | --- |
| student | 学生 |
| mentor | 导师 |
| admin | 管理员 |
| system | 系统任务 |

## 3. 权限原则

- 学生只能访问自己的资料。
- 导师只能访问分配给自己的学生。
- 管理员可访问全局数据，但关键操作必须记录日志。
- Dify 只能通过后端访问。
- 学生私有知识库不能被其他学生检索。
- 敏感字段默认脱敏展示。
- 文件访问必须经过鉴权，不能使用公开永久链接直接暴露。

## 4. 鉴权方案

MVP 推荐：

- 管理后台和导师后台使用账号密码登录。
- 学生端可先基于 NFC 卡片绑定后的短期 token 或短信验证码登录。
- API 使用 JWT 作为访问凭证。
- 后续可接入微信、短信验证码或学校统一身份。

登录后 token 内容建议：

```json
{
  "user_id": "user_001",
  "role": "mentor",
  "student_id": null,
  "mentor_id": "mentor_001"
}
```

## 5. 访问控制规则

### 5.1 学生

允许访问：

- 自己的学生档案。
- 自己的 SOP。
- 自己的任务和打卡。
- 自己的上传文件。
- 自己的 AI 问答记录。

禁止访问：

- 其他学生资料。
- 导师后台接口。
- 管理后台接口。
- Dify 原始接口。

### 5.2 导师

允许访问：

- 分配给自己的学生档案。
- 分配给自己的学生上传资料摘要。
- 分配给自己的学生 SOP。
- 分配给自己的学生目标修订。

禁止访问：

- 未分配学生资料。
- 管理员配置。
- 其他导师知识库，除非授权。

### 5.3 管理员

允许访问：

- NFC 卡片管理。
- 学生管理。
- 导师管理。
- 学校资料管理。
- Dify 知识库配置。
- 通知模板。
- 基础运营数据。

关键操作需要记录日志：

- 停用卡片。
- 分配导师。
- 修改学生状态。
- 删除或归档资料。
- 修改奖金规则。
- 修改 Dify 配置。

## 6. 敏感字段处理

| 字段 | 展示规则 |
| --- | --- |
| 手机号 | `138****0000` |
| 邮箱 | `s***@example.com` |
| 文件原始地址 | 前端不直接展示 |
| 成绩信息 | 默认仅本人、导师、管理员可见 |
| AI 问答记录 | 仅本人、导师、管理员可见 |

后端返回列表数据时默认脱敏，详情页根据权限返回完整数据。

## 7. 文件访问规则

- 文件上传后保存到对象存储或文件服务，存储桶或目录必须为私有权限。
- 数据库保存文件地址、状态和权限信息。
- 前端下载或预览文件时调用后端接口。
- 后端校验权限后生成短期访问链接或转发文件流，短期链接有效期建议不超过 10 分钟。
- 不使用永久公开文件 URL。
- 文件上传时必须进行大小、扩展名、MIME、文件头和 hash 校验。
- 文件原始名称只作为展示信息，不作为实际存储路径。
- 文件存储路径必须由后端生成，不能使用用户传入路径。
- 被拒绝或扫描失败的文件不能被下载、预览或提交 Dify。

接口：

```text
GET /api/files/{id}/preview
GET /api/files/{id}/download
```

上传前后的归属规则：

```text
激活前上传
  → owner_type = activation_session
  → 只能被当前激活会话访问

激活成功
  → owner_type = student
  → owner_id = student_id

激活会话过期
  → 文件进入待清理状态
```

## 8. Dify 权限隔离

后端调用 Dify 前必须计算可访问知识库范围：

```text
学生本人
  → 学生私有知识库 + 学校公共知识库 + 平台方法论知识库

匹配导师
  → 该学生私有知识库 + 学校公共知识库 + 平台方法论知识库 + 授权导师知识库

管理员
  → 按管理操作范围访问
```

禁止把一个学生的私有资料作为另一个学生问答的检索来源。

## 11. 异常处理

| 场景 | 处理 |
| --- | --- |
| 未登录访问 | 返回 401 |
| 越权访问 | 返回 403 |
| token 过期 | 提示重新登录 |
| 文件无权限 | 不返回文件地址 |
| 文件已被拒绝 | 不允许预览、下载或进入 Dify |
| Dify 权限范围为空 | 返回暂无可用资料 |
| 管理员关键操作失败 | 记录失败日志 |

## 12. 验收标准

- 学生不能访问其他学生资料。
- 导师不能访问未分配学生资料。
- 管理员关键操作有审计日志。
- 前端不暴露 Dify API Key。
- 文件访问必须经过后端鉴权。
- 文件上传必须经过类型、大小、文件头和归属校验。
- 文件不使用公开永久 URL。
- 激活前上传文件必须有临时归属，激活后才能绑定学生档案。
- 列表页手机号和邮箱默认脱敏。
- 学生私有资料不会进入其他学生的 AI 问答上下文。
